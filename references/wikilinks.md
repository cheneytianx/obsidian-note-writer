# 双链关键词提取指南（Wikilinks）

## 什么值得加双链

满足以下任一条件的词，加 `[[...]]`：

| 条件 | 示例 |
|------|------|
| 是一个独立概念，值得单独解释 | `[[梯度下降]]` `[[注意力机制]]` |
| 是一个工具/框架/系统 | `[[Obsidian]]` `[[Elasticsearch]]` |
| 是一个人名/组织（重要的） | `[[Andrej Karpathy]]` `[[OpenAI]]` |
| 是一个项目/系统（你在做的） | `[[支付系统重构]]` |
| 在其他笔记中已出现过 | 保持引用一致 |

## 什么不加双链

- 普通动词、形容词（"重要"、"使用"）
- 极其宽泛的词（"技术"、"系统"、"方法"）
- 只出现一次且不重要的专有名词
- **完整的句子或结论性论断**（"[[基础设施商先吃肉]]" ❌）
- **自创/临时造的词**，不在成熟文献中出现（"[[认知门槛]]" ❌ 除非后来形成独立笔记）
- **衍生/子概念**，且根概念更值得链（链 [[约束理论]] 不链 [[系统瓶颈]]）
- **来源引用型人名**（只是本次对话的信息来源，不是围绕此人形成的框架/理论）

### 加链前自检清单

提取候选词后，逐条检查。**五条全过才加双链。**

1. **一句话否决**——这是一个完整的句子/结论？→ 不加
2. **厚度测试**——点进去我能写 200 字以上的独立笔记解释它？→ 不能则不加
3. **父子关系**——它是另一个已链概念的子集/衍生？→ 链根概念，不链子概念
4. **来源 vs 概念**——这是个人名，但 ta 只是本次对话的信息来源（不是框架/理论的人格化）？→ 正文保留，不加双链
5. **自创词标记**——这是我在对话中临时造的词？→ 不加（等至少被两篇笔记独立引用再说）

## 双链书写规范

```markdown
# 标准双链
[[梯度下降]] 是训练 [[神经网络]] 的核心算法。

# 带别名（显示名与链接目标不同）
[[gradient-descent|梯度下降]] 的核心思想是...

# 链接到标题锚点
详见 [[机器学习概览#优化算法]] 章节。
```

## 提取流程

阅读对话/内容时，做两遍扫描：

**第一遍（概念扫描）**：列出所有可能的双链候选词
**第二遍（筛选）**：按上面标准去掉不值得链接的，保留 5~15 个

> 双链数量：一篇笔记 5~15 个为宜。太少失去图谱价值，太多显得杂乱。

## 自动生成的双链笔记存根（Stub）

当提取到某个重要概念但 vault 中还没有对应笔记时，写完当前笔记后，可选择性地为新概念生成一个空的 stub 笔记：

```markdown
---
tags: [stub]
created: YYYY-MM-DD
---

# 概念名

> 待补充

## 相关笔记
- [[来源笔记]]
```

询问用户是否需要生成这些 stub，不要自动创建。
